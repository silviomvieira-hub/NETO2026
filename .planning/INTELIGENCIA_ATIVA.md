## 0. Princípio Fundador

> **O candidato não pergunta. O sistema entrega.**

Neto aperta mão 14h por dia. Está cansado, no carro, no WhatsApp, sem tempo de "conversar com uma IA". Quando ele (ou o chefe de gabinete) abre a aba Inteligência, tudo já tem que estar pronto: notícias relevantes coletadas, problemas identificados, ações priorizadas, drafts de mensagem escritos. Zero digitação. Zero espera.

A aba é um **feed propositivo** que roda 24/7 em background, minerando dados externos (notícias, Diário Oficial, CLDF, TRE) e internos (demandas, visitas, leads, metas), sintetizando via Helena, e entregando **no máximo 3 ações do dia + alertas + insights invisíveis**.

Regra de ouro: nenhum card pode exigir mais de **90 segundos** de atenção. Se é complexo, o draft já vem pronto para ele só apertar "publicar" ou "enviar".

### 1.2 Regras de UX inegociáveis

1. **Tempo de leitura da aba inteira ≤ 3 minutos**
2. **Todas as ações têm draft pronto** — zero digitação do candidato
3. **3 botões padrão por card**: executar / feito / dispensar
4. **Badge numérico no sidebar** atualizado em tempo real (Supabase Realtime)
5. **Push notification** só para alertas CRÍTICOS
6. **Mobile-first obrigatório** — o candidato abre do celular em movimento
7. **Feedback loop**: cada ação executada/ignorada treina a Helena
8. **Modo "assessor"**: chefe de gabinete pode marcar "feito" em nome do candidato

### 2.2 Stack

- **Execução dos jobs**: Supabase Edge Functions (Deno) + Supabase Cron (`pg_cron`)
- **Scraping pesado**: se Deno limitar, worker Python em Railway ou Cloudflare Workers
- **LLM**: Claude Haiku 4.5 (rápido, barato) como padrão, Gemini 2.5 Flash como fallback, via skill `integracao-ia-omniroute`
- **Busca**: Google News API (free tier), RSS Parser, Playwright-Lite p/ scraping pontual
- **Cache**: Supabase KV ou `helena_cache` table (dedup por hash de URL/título)
- **Storage de embeddings** (opcional v2): `pgvector` para similaridade semântica de notícias

### 2.3 Por que Edge Functions + pg_cron

- Zero infra adicional, tudo na mesma conta Supabase
- Escalonamento automático
- Cron nativo (`pg_cron` ou Supabase scheduled functions)
- Isolamento por tenant nativo (RLS já cobre)
- Custo marginal praticamente zero no free tier para o volume do Neto

## 3. Os 5 Jobs (especificação)

### Job 1 — `news-scanner`
**Frequência:** 30 em 30 minutos
**Missão:** coletar e filtrar notícias relevantes

**Input:**
- Lista de queries (do candidato): nome dele, nomes dos 24 adversários CLDF, 10 eixos temáticos, 33 RAs
- Última execução (para buscar só o novo)

**Fontes:**
| Fonte | Método | Prioridade |
|---|---|---|
| Google News API | HTTP GET com query | ALTA |
| RSS Metrópoles | RSS Parser | ALTA |
| RSS Correio Braziliense | RSS Parser | ALTA |
| RSS G1 DF | RSS Parser | ALTA |
| RSS Agência Brasília | RSS Parser | MÉDIA |
| Feed CLDF | scraping HTML | MÉDIA |
| Diário Oficial DF | scraping diário | MÉDIA |
| Twitter/X API | API (opcional) | BAIXA |
| Nitter scraping | fallback Twitter | BAIXA |

**Pipeline:**
1. Busca em todas as fontes em paralelo
2. Deduplicação por hash (URL canônica + título normalizado)
3. Filtro de relevância:
   - **Stage 1 (regra):** match de keyword nas queries do candidato
   - **Stage 2 (LLM leve):** Claude Haiku classifica em `impacto: alto|medio|baixo` + `eixo_tematico` + `sentimento` + `acao_sugerida` (1 frase)
4. Insert em `helena_noticias_coletadas`
5. Se `impacto=critico`, dispara trigger de push notification

**Saída:** ~30-80 notícias/dia por candidato DF (estimativa)

**Custo LLM estimado:** R$ 10-25/mês por candidato (Haiku)

### Job 2 — `data-analyzer`
**Frequência:** 2 em 2 horas
**Missão:** detectar padrões, gaps e oportunidades nos dados operacionais

**Input:**
- Tabelas: `demandas`, `visitas`, `leads`, `atividades_gamificacao`, `pesquisas`, `metas_regionais`, `pessoas_campanha`

**Detectores (regras SQL puras, sem LLM):**

1. **Tendência de demanda**
   ```sql
   demandas agrupadas por (categoria, ra) nas últimas 48h
   WHERE count >= 3 AND sem_alerta_criado_ainda
   ```

2. **Queda de atividade regional**
   ```sql
   atividades_gamificacao agrupadas por ra nos últimos 7d
   COMPARADO COM média móvel dos últimos 30d
   WHERE desvio <= -15%
   ```

3. **Lead VIP cadastrado**
   ```sql
   leads inseridos últimas 24h
   WHERE ocupacao IN ('empresario', 'medico', 'lider_religioso', ...)
      OR indicado_por_lider = true
   ```

4. **Meta regional desviada**
   ```sql
   metas_regionais
   WHERE (apoiadores_atual / apoiadores_meta) < 0.8
     AND dias_restantes < 30
   ```

5. **Coordenador inativo**
   ```sql
   pessoas_campanha WHERE funcao='coordenador'
     AND ultima_atividade < NOW() - 72h
   ```

6. **Evento sem confirmações suficientes**
   ```sql
   agenda WHERE tipo='evento'
     AND data < NOW() + 48h
     AND confirmacoes < capacidade * 0.6
   ```

7. **Material em falta em RA com evento marcado**
   ```sql
   JOIN materiais + agenda + distribuicoes
   WHERE estoque_regional < necessidade_evento
   ```

**Saída:** inserts em `helena_alertas` e `helena_sinais_operacionais`

**Custo LLM:** **zero** (SQL puro)

### Job 3 — `helena-synthesizer`
**Frequência:** 4x/dia — 06:00, 12:00, 17:00, 21:00
**Missão:** consolidar tudo e gerar o briefing propositivo

**Input:**
- Todas as notícias das últimas 6h
- Todos os alertas ativos
- Resumo do data-analyzer
- Briefing anterior (para continuidade)
- Dados do candidato (posicionamento, eixos, estilo)

**Pipeline:**
1. **Agrupamento por tema** (cluster semântico via embeddings ou keywords)
2. **Seleção do comitê**: random-pick de 3 consultores lendários (dos 158) relevantes ao tema — ex: marketeiro + estrategista + cientista político
3. **Consulta ao comitê** (cada consultor é um prompt calibrado):
   - Qual ação recomenda?
   - Qual risco vê?
   - Como conecta com a campanha?
4. **Red Team** (skill `persuasao-framework`): ataca cada recomendação
5. **Priorização** via score:
   ```
   score = (impacto × 0.4) + (urgencia × 0.3) + (viabilidade × 0.2) + (alinhamento_candidato × 0.1)
   ```
6. **Top 3 ações** selecionadas
7. **Draft de execução** para cada ação (mensagem WhatsApp / post / carta / ligação script), via skill `diana-eleitoral` ou `comunicacao-persuasiva`
8. **Insert** em `helena_briefings` + `helena_acoes_recomendadas`

**Formato de output (Padrão INTEIA):**
- DECISÃO primeiro (1 frase)
- JUSTIFICATIVA (3 bullets)
- RISCO (1 frase)
- DRAFT pronto (se aplicável)

**Modelo:** Claude Haiku 4.5 (principal) / Gemini 2.5 Flash (fallback)

**Custo estimado:** R$ 50-100/mês por candidato (4 runs/dia × ~15k tokens cada)

---

### Job 4 — `opportunity-miner`
**Frequência:** 1x/dia às 05:00
**Missão:** executar a **Camada Invisível** — encontrar o que ninguém vê

**Pipeline POLARIS 5 camadas:**
1. **Superfície**: Google Trends DF (tópicos em alta)
2. **Vertical**: dados demográficos CODEPLAN, TRE-DF, CENSO
3. **Histórica**: comparação com campanhas anteriores (2018, 2022)
4. **Adversarial**: monitoramento dos 24 adversários (site, redes, notícias)
5. **Sinal fraco**: padrões estatísticos não óbvios nos dados internos

**Heurísticas:**
- Demanda crescente + zero menção pelos adversários = **janela de ocupação**
- Adversário caindo em um eixo + você forte nele = **ataque flanqueante**
- Demografia mudando em RA + infraestrutura insuficiente = **território em disputa**
- Lei nova em votação na CLDF + seu eixo = **ancoragem legislativa**

**Output:** 1 insight/dia em `helena_insights_invisiveis` com:
- Texto (máx 120 palavras)
- Janela de oportunidade (dias)
- Fontes (links)
- Plano de ação draft (opcional)
- Confiança (0-1)
- Consultores consultados

**Custo estimado:** R$ 40-60/mês por candidato (mais denso que synthesizer)

---

### Job 5 — `critical-watcher`
**Frequência:** tempo real (trigger em insert)
**Missão:** disparar push notification quando algo crítico acontece

**Triggers:**
```sql
CREATE TRIGGER helena_critico AFTER INSERT ON helena_alertas
FOR EACH ROW WHEN NEW.severidade = 'critico'
EXECUTE FUNCTION notify_push_critico();
```

**Eventos críticos:**
- Adversário cita o candidato
- Escândalo envolvendo aliado direto
- Decisão TRE/TSE afeta campanha
- Mudança de prazo legal eleitoral
- Queda abrupta em eixo-chave
- Viralização negativa em redes

**Destinatários configuráveis:**
- Candidato
- Chefe de gabinete
- Assessor de comunicação
- Coordenador jurídico (p/ alertas TRE/TSE)

**Tecnologia:** Web Push API + Service Worker (PWA)

## 4. Schema do Banco (Supabase / PostgreSQL)

```sql
-- ============================================
-- BRIEFINGS (1 por turno, 4/dia)
-- ============================================
CREATE TABLE helena_briefings (
  id              uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  tenant_id       uuid NOT NULL REFERENCES tenants(id),
  turno           text NOT NULL, -- 'manha' | 'meio_dia' | 'tarde' | 'noite'
  turno_timestamp timestamptz NOT NULL,
  resumo_executivo text NOT NULL, -- máx 280 chars
  top_3_acoes_ids uuid[],
  alertas_count   int DEFAULT 0,
  noticias_count  int DEFAULT 0,
  insight_id      uuid REFERENCES helena_insights_invisiveis(id),
  score_confianca numeric(3,2),
  modelo_usado    text,
  tokens_custo_usd numeric(6,4),
  created_at      timestamptz DEFAULT now()
);

CREATE INDEX ON helena_briefings (tenant_id, turno_timestamp DESC);

-- ============================================
-- AÇÕES RECOMENDADAS (o coração da tela)
-- ============================================
CREATE TABLE helena_acoes_recomendadas (
  id              uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  tenant_id       uuid NOT NULL REFERENCES tenants(id),
  briefing_id     uuid REFERENCES helena_briefings(id),
  prioridade      text NOT NULL, -- 'alta' | 'media' | 'baixa'
  titulo          text NOT NULL,
  justificativa   text[] NOT NULL, -- array de 3 bullets
  deadline        timestamptz,
  acao_tipo       text, -- 'ligacao' | 'post' | 'whatsapp' | 'visita' | 'reuniao' | 'publicacao'
  draft_mensagem  text,
  link_execucao   text,
  metadados       jsonb,
  status          text DEFAULT 'pendente', -- 'pendente' | 'feito' | 'ignorado'
  feedback_candidato text,
  executed_by     uuid REFERENCES auth.users(id),
  executed_at     timestamptz,
  created_at      timestamptz DEFAULT now()
);

CREATE INDEX ON helena_acoes_recomendadas (tenant_id, status, prioridade);

-- ============================================
-- ALERTAS
-- ============================================
CREATE TABLE helena_alertas (
  id              uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  tenant_id       uuid NOT NULL REFERENCES tenants(id),
  severidade      text NOT NULL, -- 'critico' | 'alto' | 'medio'
  categoria       text NOT NULL, -- 'adversario' | 'juridico' | 'operacional' | 'midia' | 'financeiro'
  titulo          text NOT NULL,
  descricao       text NOT NULL,
  fonte           text,
  link            text,
  acao_recomendada_id uuid REFERENCES helena_acoes_recomendadas(id),
  visto           boolean DEFAULT false,
  dispensado      boolean DEFAULT false,
  expires_at      timestamptz,
  created_at      timestamptz DEFAULT now()
);

CREATE INDEX ON helena_alertas (tenant_id, severidade, visto);

-- ============================================
-- NOTÍCIAS COLETADAS
-- ============================================
CREATE TABLE helena_noticias_coletadas (
  id              uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  tenant_id       uuid NOT NULL REFERENCES tenants(id),
  titulo          text NOT NULL,
  fonte           text NOT NULL,
  url             text NOT NULL,
  hash_dedup      text UNIQUE NOT NULL,
  publicado_em    timestamptz,
  resumo          text, -- 2 linhas
  eixo_tematico   text,
  impacto         text, -- 'alto' | 'medio' | 'baixo'
  sentimento      text, -- 'positivo' | 'neutro' | 'negativo'
  acao_sugerida   text,
  usado_em_post   boolean DEFAULT false,
  arquivado       boolean DEFAULT false,
  created_at      timestamptz DEFAULT now()
);

CREATE INDEX ON helena_noticias_coletadas (tenant_id, impacto, publicado_em DESC);

-- ============================================
-- INSIGHTS INVISÍVEIS (camada invisível)
-- ============================================
CREATE TABLE helena_insights_invisiveis (
  id              uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  tenant_id       uuid NOT NULL REFERENCES tenants(id),
  data            date NOT NULL,
  insight_texto   text NOT NULL, -- máx 120 palavras
  janela_oportunidade_dias int,
  fontes          jsonb, -- [{titulo, url, peso}]
  plano_acao_draft text,
  confianca       numeric(3,2),
  consultores_consultados text[],
  usado           boolean DEFAULT false,
  created_at      timestamptz DEFAULT now()
);

CREATE UNIQUE INDEX ON helena_insights_invisiveis (tenant_id, data);

-- ============================================
-- LOG DOS JOBS (observabilidade)
-- ============================================
CREATE TABLE helena_jobs_log (
  id              uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  job_name        text NOT NULL,
  tenant_id       uuid REFERENCES tenants(id),
  started_at      timestamptz NOT NULL,
  finished_at     timestamptz,
  status          text, -- 'sucesso' | 'falha' | 'parcial'
  records_in      int,
  records_out     int,
  cost_usd        numeric(6,4),
  error           text,
  metadata        jsonb
);

CREATE INDEX ON helena_jobs_log (job_name, started_at DESC);

-- ============================================
-- RLS (multi-tenant)
-- ============================================
ALTER TABLE helena_briefings ENABLE ROW LEVEL SECURITY;
-- ... repetir para todas as tabelas

CREATE POLICY tenant_isolation ON helena_briefings
  USING (tenant_id = (auth.jwt() ->> 'tenant_id')::uuid);
-- ... repetir
```

## 9. Métricas de Sucesso

### Produto
- **DAU (Daily Active Users)**: candidato + assessores abrem aba ≥2x/dia
- **Taxa de execução**: % de ações marcadas "feito" vs recomendadas — meta ≥40%
- **Tempo de leitura do briefing**: entre 60-180s (se <30s, não leu; se >300s, muito denso)
- **Notícias "arquivadas por irrelevância"**: < 20% (Helena filtrando bem)
- **Alertas críticos verdadeiros positivos**: > 85%

### Negócio
- **Demos → vendas**: meta conversão ≥30%
- **Churn pós-eleição**: < 20% (retenção em ciclos futuros)
- **NPS do candidato**: ≥ 50
- **Testemunho por nome**: pelo menos 1 candidato eleito citar Helena publicamente
