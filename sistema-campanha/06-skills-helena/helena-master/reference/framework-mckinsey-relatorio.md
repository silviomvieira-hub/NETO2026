# Framework McKinsey para Relatorios — Padrao Helena/INTEIA

> Registrado em 2026-03-01 por determinacao do fundador.
> "Nao e trabalho de escola. Comeca pelo fim. Conclusao primeiro. Numeros sao ferramenta da Helena, nao entrega pro cliente."

## Principio Fundamental: Piramide de Minto (Barbara Minto, McKinsey 1960s)

**A comunicacao e INVERTIDA em relacao ao pensamento.**
- Pensamento: dados → analise → conclusao (bottom-up)
- Comunicacao: conclusao → argumentos → evidencias (top-down)

O cliente NAO quer saber como voce chegou la. Quer saber ONDE chegou e O QUE FAZER.

## Estrutura Obrigatoria de Relatorio INTEIA

### 1. RESPOSTA (pagina 1 — o unico slide que importa)

Sumario Executivo com POSICAO:
- Conclusao principal em 1-2 frases (NAO descritiva — DECISIONAL)
- 3 achados criticos (com numero e "so what")
- Recomendacao concreta (acao + timeline + impacto esperado)
- Nivel de confianca

**Teste**: Se o decisor ler SO ESTA PAGINA e souber o que fazer, passou.
Se precisar ler mais para decidir, FALHOU.

❌ "Este relatorio analisa o perfil sociourbano da SCIA/Estrutural..."
✅ "SCIA/Estrutural e uma bomba-relogio: 8.000 jovens sem opcao, renda 4x menor que DF, e a PCDF tem 90 dias antes que o ciclo se torne irreversivel."

### 2. ARGUMENTOS (3-5 secoes — MECE)

Cada secao segue:
- **Action Title** = conclusao da secao (NAO descricao)
- **So What** = por que isso importa pro decisor
- **Evidencia** = dados que SUPORTAM a conclusao (minimo necessario)
- **Implicacao** = o que acontece se nao agir

**Action Titles — Regras:**
- Maximo 15 palavras
- E uma FRASE COMPLETA com verbo
- Contem o insight, nao a categoria
- Se alguem ler SO os titulos, entende o relatorio inteiro

❌ "Perfil Socioeconomico"
✅ "Renda media de R$1.800 cria armadilha de pobreza que alimenta 73% dos crimes"

❌ "Dados de Criminalidade"
✅ "Roubo a transeunte cresceu 340% em 3 anos — SCIA e a RA mais perigosa para pedestres"

❌ "Problemas Urbanos"
✅ "Ausencia de iluminacao em 67% das vias cria zona morta para policiamento noturno"

### 3. RESOLUCAO (recomendacoes — o que fazer)

- Acoes concretas priorizadas (urgente → importante → monitorar)
- Timeline realista
- KPIs mensuráveis
- Quem faz o que

### 4. CAMADA INVISIVEL (Helena Viu)

O diferencial. O que dados oficiais escondem. Loops, paradoxos, economia politica.
NAO e apendice — e INSIGHT PREMIUM.

### 5. APENDICE (para quem gosta de numero)

AQUI vao as tabelas cruas, censos, dados brutos. Quem quiser conferir, confere.
Mas NUNCA no corpo principal.

## Regra do "So What?"

TODA informacao no relatorio deve passar neste teste:

```
Dado: "Populacao da SCIA: 39.015 habitantes"
So what? → "So what?"
Resposta: "Nada. E um numero. Nao entra no corpo."

Dado: "Populacao da SCIA cresceu 22% mas infraestrutura ficou estagnada"
So what? → "Densidade sem infraestrutura = criminalidade. Cada 10% de crescimento
            populacional sem investimento correlaciona com +15% em roubos."
Resposta: "ISSO e insight. Entra como action title."
```

## Regra SCR (Situation-Complication-Resolution)

**Situation**: O que o decisor ja sabe (contexto minimo)
**Complication**: O problema/oportunidade que cria urgencia
**Resolution**: A recomendacao com evidencia

```
S: "SCIA/Estrutural e a 25a RA do DF, com 39 mil habitantes."
C: "Mas concentra o 3o maior IVS do DF, roubo a transeunte 340% acima da media,
    e 8.000 jovens sem perspectiva economica — enquanto a PCDF nao tem delegacia local."
R: "Tres acoes em 90 dias reduzem crimes violentos em 25-40%: delegacia movel,
    iluminacao em 12 vias criticas, programa primeiro emprego para 500 jovens."
```

## Visualizacao de Dados (McKinsey Style)

1. **Comece pelo insight** — escolha o grafico que PROVA sua conclusao
2. **Um grafico = uma mensagem** — nunca dois insights no mesmo grafico
3. **Label direto no grafico** — elimine legendas quando possivel
4. **Destaque o dado que importa** — cor de destaque no ponto critico
5. **Paleta restrita** — 2-3 cores + cinza. Vermelho so para problemas

## Tipografia de Relatorio

- **Action titles**: bold, 18-20pt, cor primaria
- **Corpo**: 14-16pt, cinza escuro, line-height 1.6
- **Dados de suporte**: 12pt, cinza medio
- **Fontes**: nunca apendice

## Checklist Anti-Trabalho-de-Escola

- [ ] Sumario executivo permite decisao sem ler mais nada?
- [ ] Action titles contam a historia sozinhos?
- [ ] Cada dado tem "so what?" explicito?
- [ ] Tabelas cruas estao no APENDICE, nao no corpo?
- [ ] Recomendacoes sao concretas (quem + o que + quando + KPI)?
- [ ] Camada Invisivel esta presente?
- [ ] Leitor ocupado entende tudo em 2 minutos?
