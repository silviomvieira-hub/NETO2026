# SKILL: Proposta Comercial 1 Página — LaTeX Premium

> **Propósito**: Gerar propostas comerciais em 1 página A4 paisagem usando LaTeX, com marca INTEIA (preto + dourado). Framework de persuasão invisível validado por 8 estudos científicos (N > 982.000 participantes).

---

## QUANDO USAR ESTA SKILL

- Cliente pediu proposta visual de 1 página para reunião comercial
- Precisa apresentar pacote de consultoria/serviço com preço, ROI e diferenciais
- Quer material impresso profissional para deixar na mesa do prospect
- Qualquer proposta INTEIA que precisa de impacto visual

## PRINCÍPIO MESTRE: PERSUASÃO INVISÍVEL

> A persuasão mais eficaz é a que o leitor sente sem identificar.
> Se o leitor consegue nomear a técnica que você está usando, você falhou.

### O Conceito

A INTEIA é consultoria de inteligência, não agência de marketing digital. A proposta transmite a confiança de um relatório McKinsey ou um pitch de private bank — dados que falam sozinhos, confiança tranquila, zero necessidade de gritar.

A ciência (Yale, Groningen, iRAISE, Chen & Hong) funciona MELHOR quando invisível. Curiosity gap via clickbait funciona para infoproduto de R$97. Para proposta de R$15k/mês, curiosity gap é criado por três números específicos que o leitor não esperava ver juntos.

### Execução Visível (PROIBIDA) vs Invisível (CORRETA)

| Técnica | Visível (proibido) | Invisível (correto) |
|---------|-------------------|-------------------|
| Curiosidade | "O que ela sabe que você não?" | Três métricas específicas sem explicação — o cérebro preenche |
| Rapport | "VOCÊ JÁ É EXCELENTE" | Usar terminologia técnica do setor com precisão |
| Reframe | "O problema não é X. É Y." (slogan) | Dados de mercado que levam o leitor a concluir sozinho |
| CTA | "Responda com 3 horários esta semana!" | "Demonstração presencial disponível — 30 minutos." |
| Antropomorfismo | "Você, eu e uma demo ao vivo" | Nome + cargo + canal direto. Sem instrução. |
| Preço | "APENAS R$15.000!!!" | Número limpo ao lado do ROI. A matemática fala. |
| Dor | "Seus problemas são..." | Dados impessoais de mercado. O leitor se reconhece. |
| Hedge | "a partir de", "contrato mínimo" | Valor direto + condições claras sem linguagem defensiva |

### Tom Proibido vs Tom INTEIA

**PROIBIDO** (marketing digital, guru, infoproduto):
- Perguntas retóricas clickbait ("O que ela sabe que você não?")
- Caixa alta emocional ("VOCÊ JÁ É EXCELENTE!")
- Linguagem de escassez/urgência artificial
- Imperativo agressivo ("Responda AGORA")
- Palavras: "poderes", "segredos", "revolucionário", "imbatível", "transformador"
- Taglines motivacionais ("Para quem já é bom e quer ser imbatível")
- Frases de coach ("O problema não é competência. É infraestrutura.")

**INTEIA** (consultoria premium, relatório técnico):
- Afirmações factuais com números específicos
- Linguagem técnica do setor do prospect (MTHFR, APOE, não "superexame")
- Calma de quem não precisa vender
- Convite, não instrução
- Vocabulário: "módulos", "protocolo", "sistema", "projeção", "capacidades"
- Taglines descritivas ("Proposta técnico-comercial")
- Observações baseadas em dados ("Infraestrutura tecnológica separa clínicas que crescem de clínicas que estabilizam.")

### Regra de Ouro

Depois de escrever qualquer texto da proposta, aplique o teste:

1. **"Isso poderia estar num anúncio de Instagram de infoproduto?"** → Se sim, reescreva.
2. **"Isso poderia estar num relatório da McKinsey?"** → Se sim, está no tom certo.
3. **"O leitor consegue identificar qual técnica de persuasão estou usando?"** → Se sim, está óbvio demais.

### Como Cada Técnica Funciona Invisível

**Curiosidade (Groningen beta=0.60)**: Não faça perguntas. Apresente 3 métricas que sozinhas fazem o leitor querer saber como. "Ticket médio de R$1.000 | Retenção de 85% | 5 horas por dia." O cérebro naturalmente pergunta "como?"

**Rapport (Yale: positivo)**: Não diga que o prospect é excelente. MOSTRE que você entende o mundo dele usando a terminologia técnica corretamente. Quando uma nutricionista lê sobre MTHFR e APOE e percebe que você sabe o que são, ISSO é rapport.

**Reframe (Yale: collaborative)**: Não declare "O problema não é X, é Y" como slogan. Apresente dados de mercado impessoais e deixe o leitor chegar à conclusão. Dados de mercado → "Consulta padrão: R$200-300, pressão de apps a R$29/mês" → o leitor pensa "preciso de infraestrutura" sem que você diga.

**CTA (Yale beta=0.379)**: Específico mas não agressivo. "Demonstração presencial com caso clínico real — 30 minutos." É específico (30 min, caso real), é ação (demonstração), mas é convite, não ordem.

**Treinamento (Chen & Hong)**: Não destaque como diferencial. Inclua "treinamento" naturalmente no cronograma de implantação. É um item da lista, não um argumento de venda.

### Estrutura Visual (3 colunas, A4 paisagem)

```
┌──────────────────────────────────────────────────────────────┐
│  HEADER: Logo INTEIA + Produto + Subtítulo técnico            │
│  Linha dourada separadora                                     │
├──────────────────────────────────────────────────────────────┤
│  HEADLINE: 3 métricas factuais separadas por pipe (|)         │
├──────────────┬───────────────────┬────────────────────────────┤
│  COLUNA 1    │  COLUNA 2         │  COLUNA 3                │
│              │                   │                            │
│  CONTEXTO    │  Módulos/         │  Fluxo Operacional         │
│  DE MERCADO  │  Capacidades      │  (linguagem técnica)       │
│  (dados      │  (lista numerada  │                            │
│  impessoais) │  com dourado)     │  Resultados Esperados     │
│              │                   │  (projeções, não promessas) │
│  PROJEÇÃO    │                   │                            │
│  FINANCEIRA  │                   │  Implantação              │
│  (números)   │                   │  (com treinamento)         │
│              │                   │                            │
│  PREÇO       │                   │  CTA                      │
│  (limpo)     │                   │  (convite, não ordem)      │
├──────────────┴───────────────────┴────────────────────────────┤
│  FOOTER: INTEIA - Endereço - Copyright                        │
└──────────────────────────────────────────────────────────────┘
```

### Proporções das Colunas
- Coluna 1: 31.5% (contexto + projeção + preço)
- Coluna 2: 34.5% (módulos/capacidades)
- Coluna 3: 31% (fluxo + resultados + implantação + CTA)

### Passo 3: Validar

Checklist obrigatório:

**Layout:**
- [ ] Cabe em 1 página A4 paisagem
- [ ] Header com texto visível (branco no preto)
- [ ] Preço em destaque com borda dourada
- [ ] Sem overflow de texto em nenhuma caixa

**Persuasão Invisível:**
- [ ] Headline são métricas factuais, NÃO pergunta retórica
- [ ] Contexto usa dados de mercado impessoais, NÃO "você é excelente"
- [ ] Módulos usam terminologia técnica do setor, NÃO "poderes"
- [ ] CTA é convite específico, NÃO instrução imperativa
- [ ] Sem hedge ("mínimo", "a partir de", "dependendo")
- [ ] Cronograma inclui treinamento naturalmente
- [ ] TESTE: "Isso parece anúncio de Instagram?" → Se sim, reescrever

## REGRAS TÉCNICAS CRÍTICAS

### 1. NUNCA usar TikZ com alturas fixas para caixas de conteúdo
```latex
% ❌ ERRADO — altura fixa, texto transborda
\begin{tikzpicture}
\fill[red!8] (0,0) rectangle (\textwidth,-24mm);
\node[...] { texto longo };
\end{tikzpicture}

% ✅ CORRETO — tcolorbox se adapta ao conteúdo
\begin{tcolorbox}[box, colback=danger!6, colframe=danger!6]
texto longo
\end{tcolorbox}
```

### 2. Header via tcolorbox (NÃO TikZ overlay)
```latex
\begin{tcolorbox}[enhanced, sharp corners, boxrule=0pt,
  width=\paperwidth, enlarge left by=-8mm, enlarge right by=-8mm,
  colback=headerbg, colframe=headerbg,
  borderline south={1.2mm}{0pt}{gold}]
  % conteúdo do header
\end{tcolorbox}
```

### 3. Background via TikZ overlay (só fill, sem nodes)
```latex
\begin{tikzpicture}[remember picture, overlay]
  \fill[bglight] (current page.south west) rectangle (current page.north east);
\end{tikzpicture}
```

### 4. Fontes e tamanhos

| Elemento | Tamanho | Leading |

| Título principal (INTEIA Xxx) | 24pt | 26pt |
| Subtítulo do produto | 12.5pt | 14pt |
| Headline (métricas) | 11pt | 13pt |
| Títulos de seção | 10pt | 12pt |

| Texto de descrição | 6.8-7.2pt | 8.5-9.5pt |

| Footer | 5.5pt | 7pt |

### 5. Espaçamentos

| Entre | Valor |
|-------|-------|
| Header → Headline | 1.5mm |
| Headline → Colunas | 3.5mm |
| Entre seções (coluna) | 3mm |
| Entre itens dentro de caixa | 1-1.5mm |
| Padding das caixas | 2.5-3mm |

### 6. Se compilar com 2 páginas
1. Reduzir descrições dos módulos (frases mais curtas)
2. Comprimir leading (11→10.5, 9.5→8.5)
3. Reduzir espaço entre itens (2mm→1.5mm)
4. Reduzir padding das caixas (3.5mm→3mm)

## PROCESSO DE GERAÇÃO (PASSO A PASSO)

1. **Receber briefing** — setor, capacidades do sistema, preço, ROI estimado
2. **Montar headline com 3 MÉTRICAS** — números factuais que criam curiosidade naturalmente
3. **Escrever subtítulo TÉCNICO** — descritivo, sem motivacional
4. **Contexto de mercado com DADOS IMPESSOAIS** — taxas, preços, pressões do setor
5. **Última frase do contexto = observação** (não slogan) — ex: "Infraestrutura tecnológica separa clínicas que crescem de clínicas que estabilizam."
6. **Projeção financeira com números LIMPOS** — "Hoje" vs "Com INTEIA", sem exclamação
7. **Preço DIRETO** — valor + condições, sem hedge, sem justificativa
8. **Módulos com TERMINOLOGIA DO SETOR** — rapport invisível via domínio técnico
9. **Fluxo OPERACIONAL** — passos técnicos, linguagem profissional
10. **Resultados como PROJEÇÕES** — "projetado", "esperado", não "garantido"
11. **Cronograma com treinamento NATURAL** — item da lista, não argumento
12. **CTA como CONVITE** — específico (30 min, caso real) mas calmo
13. **Compilar** — `pdflatex -interaction=nonstopmode proposta-NOME.tex`
14. **Teste McKinsey** — "Isso parece Instagram?" Se sim, reescrever
